# Task manager

## About 'Task manager'

This is a task manager, written in and for educational and demonstrational purposes.

A full-stack SPA task manager for organising personal tasks with role-based access, real-time search with debounce, filtering, and pagination. Users can register, log in, create tasks, filter, and manage their personal todo list. Administrators have full visibility and control over all tasks. Built with Laravel and Nuxt.

## Key Features

- Secure token-based authentication (Laravel Sanctum API tokens),
- Role-based access control: admin and regular user (many-to-many roles),
- Full CRUD for tasks with validation, policies, and service layer,
- Paginated task list with filtering by status,
- Search with debounce (300ms) and query parameter synchronisation,
- Responsive design with dark/light theme support,
- Reusable UI component library (buttons, inputs, links, etc.),
- FormRequests and DTOs on the backend,
- Strict TypeScript types on the frontend,
- API documentation generated by Scribe,
- Dockerised development environment with a single startup command,
- Automatic frontend build via Docker.

## Requirements

- Docker Desktop (or Docker Engine + Docker Compose).
- Windows / macOS / Linux with WSL2 recommended for Windows users.

## Getting Started

- Clone the repository:
``` bash
git clone [repository-url]
```

- Change directory to project:
``` bash
cd /path/to/task-manager/
```

- Install PHP dependencies locally (this provides the `sail` command):
``` bash
composer install
```

- Create .env file:
``` bash
cp .env.example .env
```

- Run Docker Desktop (with wsl - for Windows only)

- Run wsl (for Windows only):
``` bash
wsl
```

- Start Docker containers (Laravel, Nuxt, Nginx):
``` bash
./vendor/bin/sail up -d
```

- Add new tab in terminal and enter the Laravel container:
``` bash
docker exec -it task-manager-laravel.test-1 bash
```

- Install php dependencies (optional, if missing some required php extension in base system):
``` bash
composer install
```

- Generate application key inside the container:
``` bash
php artisan key:generate
```

or outside the container

``` bash
./vendor/bin/sail artisan key:generate
```

- Run migrations with seeds:
``` bash
php artisan migrate --seed
```

or

``` bash
php artisan migrate:fresh --seed
```

- The Nuxt frontend is already started automatically inside the same container (in the `frontend` folder). If you need to manually build the frontend (e.g. after updating packages), inside the container run:
``` bash
cd frontend
npm install
npm run build
```

- In browser go to http://localhost/

That's it! Thank you!

## API Documentation

Full interactive API documentation is available at http://localhost/docs.

To generate or update the documentation, enter the Laravel container and run:
``` bash
docker exec -it task-manager-laravel.test-1 bash
php artisan scribe:generate
```

Alternatively, you can use the predefined Composer script:
``` bash
composer openapi:generate
```
